## Reflexión Final: DynamoDB en el Ecosistema NoSQL

### Comparativa: ¿Qué diferencias clave notaste entre trabajar con DynamoDB y lo que has visto de MongoDB? (Piensa en la sintaxis de inserción, las consultas, el esquema…).
La principal diferencia que noté entre DynamoDB y MongoDB es la forma en la que se manejan los datos. En DynamoDB todo va con un formato más “estructurado” tipo JSON pero con tipos explícitos (por ejemplo "S" para string o "N" para número), mientras que en MongoDB simplemente creo que se usan documentos JSON normales. Además, en DynamoDB las consultas suelen depender mucho de las claves primarias y los índices secundarios, mientras que en MongoDB se pueden hacer búsquedas más libres con filtros. También noté que DynamoDB no tiene un esquema fijo, pero sí te obliga a definir una clave de partición y, a veces, una de orden, mientras que MongoDB es más flexible con eso.
